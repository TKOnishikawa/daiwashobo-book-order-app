# 注文書ジェネレーター 仕様書

## 1. 目的

大和書房 営業部が書店向け注文書（重版・新刊・フェア案内）を、ISBN入力→自動補完→Word/PDF出力の流れでブラウザから即座に生成できるようにする。現在のExcel/Word手作業（15-30分/枚）を80%削減し、フォーマットの属人化・転記ミスを解消する。

## 2. スコープ / 非スコープ

### スコープ（MVP）
- ISBN-13入力による書籍情報自動入力（book_master.json参照）
- 注文書フォーム（13フィールド + 売行き実績テーブル + 画像アップロード）
- A4リアルタイムプレビュー
- Word(.docx)自動生成・ダウンロード
- PDF印刷（window.print + @media print CSS）
- 入力データのlocalStorage保存・呼出・削除
- JSON エクスポート/インポート（PC間データ移動）
- GitHub Pagesへの静的デプロイ

### 非スコープ（将来フェーズ）
- 書籍マスタの自動更新（Sheets連携 or API取得）
- 書影のWord内埋め込み（現在はプレビューのみ）
- 複数書籍の一括Word生成
- kintone連携（受発注管理）
- POS実績の自動引用
- ユーザー認証・権限管理
- バックエンド / データベース

## 3. ユーザー

| ユーザー | 役割 | ITリテラシー | 利用頻度 |
|---------|------|-------------|---------|

## 4. 技術スタック

| 層 | 技術 | 理由 |
|----|------|------|
| フレームワーク | Next.js 16（静的エクスポート） | プロトタイプ既存。SSR不要 |
| UI | React 19 (Client Components) | 既存実装 |
| Word生成 | docx@9.5.0 + file-saver | ブラウザ完結、サーバー不要 |
| データ | book_master.json（public/直下） | 静的。将来API化可能 |
| 保存 | localStorage + JSON DL/UL | サーバーレス。オフライン可 |
| デプロイ | GitHub Pages + Actions | 無料、dx-projectsと同パイプライン |

## 5. 画面設計

### 画面一覧
| 画面 | 種別 | 主要操作 |
|------|------|---------|
| メイン画面 | 2パネル（フォーム + プレビュー） | ISBN検索、フォーム入力、Word DL、PDF印刷 |
| 保存リスト（モーダル or サイドバー） | リスト | 保存データの選択・削除・インポート |

### 画面遷移
```
[メイン画面]
  ├→ ISBN入力 → 自動補完 → フォーム入力 → プレビュー確認
  ├→ Word DL / PDF印刷
  ├→ 保存ボタン → 名前入力 → localStorage保存
  ├→ 呼出ボタン → 保存リスト → 選択 → フォーム復元
  └→ 新規作成 → フォームリセット
```

## 8. 保存機能仕様

### localStorage構造
```typescript
interface SavedOrder {
  id: string;           // crypto.randomUUID()
  name: string;         // ユーザー命名（例: "夢をかなえるゾウ 3刷"）
  createdAt: string;    // ISO 8601
  updatedAt: string;    // ISO 8601
  form: OrderFormData;
  salesData: SalesRow[];
}

// localStorage key: "book-order-saved"
// value: SavedOrder[]（JSON.stringify）
```

### 操作
| 操作 | トリガー | 動作 |
|------|---------|------|
| 保存 | 保存ボタン | 名前入力→localStorage書込 |
| 上書き保存 | 保存ボタン（既存データ選択中） | 確認ダイアログ→上書き |
| 呼出 | 呼出ボタン→リスト選択 | フォーム+売行きデータを復元 |
| 削除 | リスト内の削除ボタン | 確認ダイアログ→削除 |
| エクスポート | JSONダウンロードボタン | 全保存データをJSONファイルでDL |
| インポート | JSONアップロード | ファイル選択→マージ or 上書き |

## 9. 受入条件（Acceptance Criteria）

### Must（これがないと完成と言えない）
- [ ] ISBN入力で書籍情報が自動入力される
- [ ] フォーム入力内容がリアルタイムプレビューに反映される
- [ ] Word DLで注文書が生成される
- [ ] **Word出力が実物の注文書フォーマットと目視一致する**
- [ ] GitHub Pages URLからアクセスできる
- [ ] 入力データをlocalStorageに保存・呼出できる

### Should（MVPでは省略可）
- [ ] PDF印刷がA4に正しく収まる
- [ ] 必須フィールド未入力時に警告を表示する
- [ ] JSON エクスポート/インポートでPC間データ移動ができる
- [ ] フォームリセット（新規作成）機能がある
- [ ] 使い方ガイド（ツールチップ or ヘルプ）がある

### Nice（後回しでOK）
- [ ] Dev機能（座標表示、ドラッグモード）が本番で非表示
- [ ] OGP / favicon設定
- [ ] 書影のWord内埋め込み

## 10. ガードレール設計

| 層 | 設定 |
|----|------|
| L1: 行動制限 | クライアントサイド完結。外部API呼出なし。データは端末内のみ |
| L2: 出力検証 | Word生成前に必須フィールド（タイトル・著者）の存在チェック |
| L3: 予算制御 | ホスティング$0（GitHub Pages）。API課金なし |
| L4: 人間承認 | 長島さんのWord出力フォーマット承認（Sprint 0） |

## 11. リスクと対策

| リスク | 確率 | 対策 |
|-------|------|------|
| Word出力が実物と合わない | 高 | Sprint 0で実物突合を最優先 |
| 書籍マスタが古い | 高 | 手動更新手順を文書化。Phase 2でSheets連携 |
| 営業担当が使いこなせない | 中 | ハンズオン + アプリ内ガイド |
| GitHub PagesのURL制限（社内プロキシ） | 中 | 動作確認後、必要ならVercelに切替 |
