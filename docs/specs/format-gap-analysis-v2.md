# 注文書フォーマット GAP分析 v2 — docx実データ検証版

> **作成日**: 2026-03-19
> **検証方法**: docxファイルのXMLを直接パースしフォント・サイズ・色を正確に抽出
> **対象**: 添付スクショ（資本主義と、生きていく。）+ docxサンプル（大島梢絵「私たちの読書生活」）

## フォントマップ（docxから抽出した実際の使用フォント）

| 用途 | フォント | 備考 |
|------|---------|------|
| バナー・キャッチコピー | **HG創英角ﾎﾟｯﾌﾟ体** | 等幅版。装飾的なキャッチ向け |
| ジャンルタグ・拡材 | **HGS創英角ﾎﾟｯﾌﾟ体** | プロポーショナル版 |
| FAX行・著者名・累計実績 | **HGP創英角ｺﾞｼｯｸUB** | 極太ゴシック、本文向け |
| FAX番号 | **ＭＳ ゴシック** | 等幅。数字の並び向け |
| 注文欄・ISBN等 | **HGS創英角ｺﾞｼｯｸUB** | 太ゴシック |

### ツッコミ①: フォントが3系統ある
ユーザー指示は「HGP創英角ｺﾞｼｯｸUB」「HGS創英角ｺﾞｼｯｸUB」だが、実物docxではキャッチコピーに**HG創英角ﾎﾟｯﾌﾟ体**（ポップ体）を使っている。添付スクショの「発売2週間で、3刷決定！！」も同じポップ体に見える。
**→ キャッチコピーのフォントを要確認。ポップ体にするか、ゴシックUBに統一するか？**

### ツッコミ③: FAX行のサイズ差が最大の再現度ボトルネック
プロトタイプでは全体が「小さな補足情報」扱いだが、実物では**22ptの太字で主張が強い**。
A4プレビューの限られた幅で22ptを再現するとFAX番号が折り返す可能性あり。
**→ rem換算を実物比率で計算し直す必要がある**

### GAP-E: タイトルエリア（緑背景ブロック）

添付スクショでは8/9/10/11がすべて**緑背景**に入っている。

| 属性 | 添付スクショ | 現プロトタイプ | GAP |
|------|------------|-------------|-----|
| 背景 | **緑背景**（#1B5E20系）でタイトル〜著者行を囲む | 背景なし（白） | ★★★ |
| タイトルフォント | HGP創英角ｺﾞｼｯｸUB（推定）、白文字 | sans-serif、黒文字 | ★★★ |
| タイトルサイズ | 添付から推定 ≈ 28-32pt | 1.5rem ≈ 18px ≈ **13.5pt** | ★★★ |
| サブタイトル | 緑背景内、白文字、タイトルより小さめ | 黒文字、背景なし | ★★ |
| 著者名 | `品川皓亮` 極太・大きめ（推定20-22pt） | 0.88rem ≈ **10pt** | ★★★ |
| 著者肩書き | `（株）COTEN 歴史調査チーム` 小さめ | なし（フィールド自体がない） | ★★ |
| 判型・頁・価格 | `四六判 312頁 本体1800円` 著者の右 | あり | ✅ 配置は合ってる |

### ツッコミ④: 緑背景ブロックが最大のデザイン差分
実物はタイトル〜著者行を**緑背景で1つのブロック**として囲んでいる。プロトタイプは白背景にバラバラの要素。これは**レイアウト構造の変更**が必要で、CSS修正だけでは済まない。
**→ of-title 〜 of-author-line を div で囲み、背景色を適用する方針**

### GAP-F: サイズ比率の全体的なずれ

A4（210mm × 297mm）上での実物のフォントサイズ比率:

| 要素 | 実物pt | プロトタイプrem | プロト換算pt | **倍率差** |
|------|--------|-------------|-----------|----------|
| バナー | 22pt | 1.15rem | ~12pt | **1.8x** |
| キャッチ | 26pt | 0.92rem | ~11pt | **2.4x** |
| FAX「営業部」 | 22pt | 0.82rem | ~10pt | **2.2x** |
| タイトル | ~30pt | 1.5rem | ~14pt | **2.1x** |
| 著者名 | ~22pt | 0.88rem | ~10pt | **2.2x** |

### ツッコミ⑤: 全要素が約2倍小さい
プロトタイプは**実物の約50%のサイズ**で統一的にスケールダウンしている。これは `preview-panel` の幅（680px）がA4実寸（約794px @96dpi）より小さいため。
**→ 2つのアプローチがある**:
- **(A) プレビューパネルを拡大** → 画面に収まらない可能性
- **(B) rem基準値を1.8-2.0倍に** → 他の要素とのバランス崩壊リスク
- **(C) 各要素個別にサイズ調整** → 手間だが最も正確

**推奨: (C)。各要素のrem値を実物比率に合わせて個別調整。**
